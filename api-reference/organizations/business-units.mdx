## Overview

Business units represent organizational subdivisions such as departments, cost centers, or regional offices. They support hierarchical structures and are used for:

- Approval routing
- Budget allocation
- Access control
- Reporting segmentation

```bash
curl https://back.flowieapp.com/organizations/org_abc123/business-units \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Create Business Unit

```
POST /organizations/{id}/business-units
```

### Request Body

<ParamField body="name" type="string" required>
  Business unit name
</ParamField>

<ParamField body="code" type="string">
  Short code (e.g., "FIN", "OPS")
</ParamField>

<ParamField body="parentId" type="string">
  Parent business unit ID for hierarchy
</ParamField>

<ParamField body="managerId" type="string">
  User ID of the business unit manager
</ParamField>

### Example

```bash
curl -X POST https://back.flowieapp.com/organizations/org_abc123/business-units \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Marketing",
    "code": "MKT",
    "parentId": null,
    "managerId": "user_123"
  }'
```

## Update Business Unit

```
PUT /organizations/{id}/business-units/{buId}
```

## Delete Business Unit

```
DELETE /organizations/{id}/business-units/{buId}
```

<Warning>
  Deleting a business unit will reassign its members and documents to the parent unit (or organization root).
</Warning>
