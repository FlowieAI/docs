## API Availability

<Warning>
  **API creation endpoints are coming soon.** Currently, the API supports read operations only. Creating invoices, purchase orders, and other documents is done through the Flowie web application or file imports.
</Warning>

## Base URL

All API requests should be made to the main backend gateway:

```
https://back.flowieapp.com
```

For staging/testing:

```
https://back-staging.flowieapp.com
```

## Service Endpoints

Each service has a specific path prefix:

| Service | Path | Description |
|---------|------|-------------|
| **Organizations** | `/organizations` | Organization and business unit management |
| **Documents** | `/doc` | Document upload, storage, and retrieval |
| **Budget** | `/budget` | Budget creation and transaction tracking |
| **Purchase** | `/purchase` | Purchase requests and orders |
| **Invoices** | `/invoices` | Invoice management (v3) |
| **Tags** | `/tag` | Tag groups and object tagging |

### Example Requests

```bash
# List organizations
GET https://back.flowieapp.com/organizations

# Get budget details
GET https://back.flowieapp.com/budget/organizations/{org_id}/budgets/{budget_id}

# List documents
GET https://back.flowieapp.com/doc/organizations/{org_id}/documents
```

## Authentication

All requests require Bearer token authentication:

```bash
curl https://back.flowieapp.com/invoices \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

See [Authentication](/authentication) for details on obtaining tokens.

## Request Format

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | Bearer token |
| `Content-Type` | Yes (POST/PUT) | `application/json` |
| `Accept` | No | `application/json` |

### Pagination

List endpoints support pagination:

```bash
GET /invoices?page=1&per_page=20
```

Response includes pagination metadata:

```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 150,
    "total_pages": 8
  }
}
```

### Filtering

Most list endpoints support filtering:

```bash
GET /invoices?status=pending_approval&supplier_id=sup_xyz
```

### Sorting

Sort results using the `sort` parameter:

```bash
GET /invoices?sort=-created_at  # Descending
GET /invoices?sort=amount       # Ascending
```

## Response Format

### Success Response

```json
{
  "data": {
    "id": "inv_abc123",
    "number": "INV-2024-001",
    ...
  }
}
```

### Error Response

```json
{
  "error": {
    "code": "validation_error",
    "message": "Invalid invoice number format",
    "details": [
      {
        "field": "number",
        "message": "Must match pattern INV-YYYY-NNN"
      }
    ]
  }
}
```

## HTTP Status Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing token |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found |
| 422 | Validation Error |
| 429 | Rate Limited |
| 500 | Server Error |
